# Game studio subagents

Fourteen specialist roles that collaborate to build Dart/Flutter mobile games with the
`dart-mobile-game-studio` skill. **This folder is the canonical source of truth.** Tool-specific
copies are generated by `sync-agents.py`:
- **Claude Code:** `.claude/agents/<name>.md` (native subagents — invoke by name or let Claude route).
- **Cursor:** `.cursor/rules/agents/<name>.mdc` (per-role rules / custom-mode prompts).
- **Codex:** these files + the "Subagents" section in the repo-root `AGENTS.md`.

## The team (14 specialists)
**Build roles**
| Agent | Role |
|---|---|
| `game-coordinator` | PM: decompose the task, sequence work, decide who's next (first) |
| `game-designer` | mechanics, core loop, progression, economy concept, Mini-GDD; mode + template |
| `engine-architect` | mode (Flutter-widgets / Flame / hybrid), pure-Dart-core architecture, state mgmt, perf budget |
| `gameplay-programmer` | implement systems, abilities, interaction, UI flow in Dart/Flutter/Flame |
| `art-director` | original game art as code: CustomPainter/SVG, Flame sprites/atlases, palettes, licensing |
| `narrative-writer` | quests, copy, lore, tutorial flow, localized ARB strings; explains decisions to the team |
| `balance-economist` | difficulty/progression curves, economy, win-rate/tempo; tuning as JSON |
| `qa-tester` | unit (dart test), widget (flutter_test), Patrol E2E, edge cases, accessibility |

**Review, audit & release roles** (audits are read-only — report findings, route fixes back)
| Agent | Role |
|---|---|
| `code-reviewer` | reviews a single diff/PR: Dart bugs, style, architecture vs the skill + quality bar |
| `code-auditor` | sweeps the whole codebase: systemic correctness, consistency, dead code, coverage |
| `security-auditor` | data leaks, insecure storage/network, secrets, permissions, kids-privacy (both stores) |
| `performance-auditor` | 60 fps, jank, rebuilds, image size, leaks, Flame lifecycle; DevTools plan |
| `legal-compliance` | App Store + Google Play policies, Kids/Families, COPPA/GDPR-K, package & asset licensing |
| `release-engineer` | App Store **and** Google Play submission readiness (icons, signing, metadata, testing) |

## Pipeline
```
coordinator → designer → architect → ┬─ gameplay-programmer ─┐→ qa-tester → code-reviewer ──┐
                                      ├─ art-director         │  (loop back on failures)     │
                                      ├─ narrative-writer     │                              ▼
                                      └─ balance-economist    ┘   pre-release audit gate:
                                                                  code-auditor · security-auditor ·
                                                                  performance-auditor · legal-compliance
                                                                            │
                                                                            ▼
                                                                  release-engineer → submit (both stores)
```

## How to invoke
- **Claude Code:** `@game-designer …` or describe the need — Claude auto-routes by `description`.
  Subagents can't spawn each other, so `game-coordinator` returns a plan to execute step by step.
- **Cursor:** open the matching rule in `.cursor/rules/agents/`, or paste it as a Custom Mode prompt.
- **Codex:** "act as the <role> agent" — it reads the spec here via `AGENTS.md`.

## Editing
Edit the files **here**, then regenerate the tool copies:
```bash
.agents/agents/sync-agents.py            # write .claude/agents + .cursor/rules/agents
.agents/agents/sync-agents.py --check    # CI: fail if any copy drifted
```

## Shared contract (every agent enforces)
No copyrighted assets · testable pure-Dart core (no `package:flutter` import) · accessibility via
`Semantics` · kids safety & privacy for **both** Apple Kids and Google Play Families · the Dart
quality bar in `../skills/dart-mobile-game-studio/references/dart/README.md` and
`references/quality-policy.md` · package choices via `references/package-policy.md` · **no store-
compliance guarantees** (checklists and risks only).
